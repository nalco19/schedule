AGENDA DE FÉRIAS — GUIA MUITO SIMPLES

O que está aqui:
- index.html = a página da app
- style.css = o visual
- app.js = a lógica da app
- data.js = os dados iniciais da sua agenda

COMO PUBLICAR NO GITHUB PAGES (SEM TERMINAL)

1) Crie uma conta no GitHub
2) Clique em "New repository"
3) Dê o nome: agenda-ferias
4) Deixe o repositório como Public
5) Clique em "Create repository"

6) Dentro do repositório, clique em "Add file" > "Upload files"
7) Arraste ESTES 4 ficheiros para o GitHub
8) Clique em "Commit changes"

9) Vá a "Settings" > "Pages"
10) Em "Build and deployment", em "Source", escolha "Deploy from a branch"
11) Em "Branch", escolha "main" e a pasta "/ (root)"
12) Clique em "Save"

13) Espere um pouco e o GitHub vai dar-lhe o link do site

COMO EDITAR RAPIDAMENTE
- Abra o site
- Escreva nas caixas
- Clique em "Guardar"

COMO ALTERAR A VERSÃO OFICIAL DO SITE
- No GitHub, abra o ficheiro data.js
- Clique no ícone do lápis
- Troque os textos
- Clique em "Commit changes"

NOTA IMPORTANTE
- O botão "Guardar" guarda as alterações só neste navegador/dispositivo
- Se quiser que a versão publicada mude para todos os dispositivos, altere o ficheiro data.js no GitHub

DICA
Se preferir, também pode instalar GitHub Desktop (grátis), mas não é obrigatório.
